移动端对话的场景往往是「路上改文案」「排队时查资料」。屏幕小、输入法切换频繁，Prompt 要更短、更结构化。

## 移动端入口选择

1. **手机浏览器**：Chrome / Safari 访问 `chat.deepseek.com`，可添加到主屏幕。
2. **微信内置浏览器**：部分链接会在微信内打开，注意登录态可能与系统浏览器不共享。
3. **官方 App**（若有提供）：通常通知与登录更稳定，适合高频用户。

## 适合手机的 Prompt 写法

- 一条消息只做一个子任务
- 用数字列表代替长段落
- 先让模型给「3 条要点」，满意再展开

示例（通勤时改标题）：

```
把下面标题改成 5 个版本，每个不超过 18 字，风格偏干货：
[粘贴原标题]
```

## 语音输入注意事项

- 说完后检查识别错误，尤其是专业术语
- 英文产品名用「英文原词 + 中文说明」
- 语音适合头脑风暴，定稿仍建议打字确认约束

## 弱网与中断处理

- 发送长消息前确认网络稳定
- 若回复中断，发「请从上一段第 3 点继续，不要重复前文」
- 重要结果及时复制到备忘录或导出（见 [导出教程](/posts/deepseek-chat-export-share/)）

## 移动端常见问题

| 问题 | 处理 |
|------|------|
| 页面空白 | 换浏览器或清缓存后重登 |
| 键盘遮挡输入框 | 横屏或缩短单次输入 |
| 回答被截断 | 要求「分两段输出，先输出上半部分」 |
