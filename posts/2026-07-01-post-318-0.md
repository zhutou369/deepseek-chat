## 创建企业通讯平台机器人：钉钉与企业微信

为了让 DeepSeek AI 能够与员工互动，我们需要在钉钉或企业微信中创建一个机器人，充当 AI 的接口。

### 钉钉自定义机器人创建

1.  **进入群聊：** 在你希望机器人服务的钉钉群中，点击右上角的群设置图标。
2.  **添加机器人：** 选择“智能群助手” > “添加机器人” > “自定义”。
3.  **配置机器人：**
    *   **机器人名字：** 填写一个易于识别的名称，如“DeepSeek 知识助手”。
    *   **安全设置：** 务必选择“签名（Sign）”或“IP 地址段”至少一种，推荐“签名（Sign）”以增强安全性。如果你选择“签名”，系统会为你生成一个 `Sign` 密钥，请务必记录下来。
    *   **添加：** 完成配置后，系统会生成一个 `Webhook` 地址。**这个地址和签名密钥是机器人接收消息的关键，请务必保存。**

### 企业微信应用创建与机器人配置

企业微信的机器人功能通常通过“自建应用”实现，功能更为强大和灵活。

1.  **进入管理后台：** 登录企业微信管理后台（work.weixin.qq.com）。
2.  **创建自建应用：** 导航到“应用管理” > “自建” > “创建应用”。
    *   **应用名称：** 填写“DeepSeek 知识库”或其他自定义名称。
    *   **可见范围：** 选择机器人能服务的部门或成员。
    *   **创建完成后，记录下 `AgentId` 和 `Secret`。**
3.  **配置接收消息服务器：**
    *   在应用详情页，点击“接收消息”设置中的“设置 API 接收消息”。
    *   你需要提供一个 URL（即你的 Flask 应用的公网地址）、`Token` 和 `EncodingAESKey`。
        *   `Token` 和 `EncodingAESKey` 可以自行生成或使用系统推荐的，同样需要妥善保管。
        *   **URL:** 在本地测试时，可以使用 `ngrok` 将本地 Flask 服务暴露到公网。
    *   完成配置后，企业微信会将用户的消息推送到你提供的 URL。

## 核心集成逻辑：Flask 应用搭建

现在，我们来编写 Python 代码，搭建一个 Flask 应用来处理来自钉钉/企业微信的消息，并调用 DeepSeek API 进行回复。

### 1. Flask 应用骨架

```python
from flask import Flask, request, jsonify
import requests
import json
import hashlib
import base64
import hmac
import time
import os
from deepseek import Deepseek

app = Flask(__name__)

# 配置 DeepSeek API
DEEPSEEK_API_KEY = os.getenv("DEEPSEEK_API_KEY", "YOUR_DEEPSEEK_API_KEY")
deepseek_client = Deepseek(api_key=DEEPSEEK_API_KEY)

# 钉钉配置
DINGTALK_WEBHOOK_URL = os.getenv("DINGTALK_WEBHOOK_URL", "YOUR_DINGTALK_WEBHOOK_URL")
DINGTALK_SIGN_SECRET = os.getenv("DINGTALK_SIGN_SECRET", "YOUR_DINGTALK_SIGN_SECRET")

# 企业微信配置 (仅作为示例，实际集成需要更多细节，如消息解密等)
WECOM_AGENT_ID = os.getenv("WECOM_AGENT_ID", "YOUR_WECOM_AGENT_ID")
WECOM_SECRET = os.getenv("WECOM_SECRET", "YOUR_WECOM_SECRET")
# WECOM_TOKEN 和 WECOM_ENCODING_AES_KEY 在接收消息时用于验证和解密
# 这里暂时省略企业微信消息的解密和验证逻辑，聚焦DeepSeek集成

# 存储企业内部知识的简单文本文件
KNOWLEDGE_FILE = "enterprise_knowledge.txt"

def load_knowledge_base():
    """从文件中加载企业知识库内容"""
    if os.path.exists(KNOWLEDGE_FILE):
        with open(KNOWLEDGE_FILE, "r", encoding="utf-8") as f:
            return f.read()
    return ""

# 加载知识库
enterprise_knowledge = load_knowledge_base()

def generate_dingtalk_sign(timestamp, secret):
    """生成钉钉签名"""
    string_to_sign = f'{timestamp}\n{secret}'
    hmac_code = hmac.new(secret.encode('utf-8'), string_to_sign.encode('utf-8'), digestmod=hashlib.sha256).digest()
    return base64.b64encode(hmac_code).decode('utf-8')

def send_dingtalk_message(content):
    """发送消息到钉钉"""
    timestamp = str(round(time.time() * 1000))
    sign = generate_dingtalk_sign(timestamp, DINGTALK_SIGN_SECRET)
    
    headers = {
        "Content-Type": "application/json;charset=utf-8"
    }
    data = {
        "msgtype": "text",
        "text": {
            "content": content
        },
        "at": {
            "isAtAll": False
        }
    }
    
    webhook_url_with_sign = f"{DINGTALK_WEBHOOK_URL}&timestamp={timestamp}&sign={sign}"
    response = requests.post(webhook_url_with_sign, headers=headers, data=json.dumps(data))
    return response.json()

def call_deepseek_api(user_query, context=""):
    """调用 DeepSeek API 获取回复"""
    messages = [
        {"role": "system", "content": "你是一个专业的企业知识问答机器人。请根据提供的知识库内容，简洁准确地回答用户问题。如果知识库中没有相关信息，请告知用户并建议寻求其他帮助。"},
    ]
    if context:
        messages.append({"role": "system", "content": f"企业知识库内容：\n{context}"})
    
    messages.append({"role": "user", "content": user_query})

    try:
        response = deepseek_client.chat.completions.create(
            model="deepseek-chat", # 或 deepseek-coder
            messages=messages,
            stream=False
        )
        return response.choices[0].message.content
    except Exception as e:
        print(f"调用 DeepSeek API 失败: {e}")
        return "很抱歉，DeepSeek API 调用失败，请稍后再试。"

@app.route('/webhook/dingtalk', methods=['POST'])
def dingtalk_webhook():
    data = request.json
    msg_type = data.get("msgtype")
    
    if msg_type == "text":
        content = data["text"]["content"].strip()
        sender_nick = data["senderNick"]
        
        print(f"收到来自钉钉的消息: {sender_nick} - {content}")

        # 将企业知识库内容作为上下文发送给DeepSeek
        response_text = call_deepseek_api(content, enterprise_knowledge)
        send_dingtalk_message(response_text)
        
    return jsonify({"errmsg": "ok", "errcode": 0})

@app.route('/webhook/wecom', methods=['POST', 'GET'])
def wecom_webhook():
    # 企业微信消息验证 (GET请求)
    if request.method == 'GET':
        # 实际生产中需要验证 signature、timestamp、nonce 和 echostr
        # 这里仅为简化示例，直接返回 echostr
        echostr = request.args.get('echostr')
        return echostr

    # 接收并处理企业微信消息 (POST请求)
    if request.method == 'POST':
        # 实际生产中需要解密消息体
        xml_data = request.data.decode('utf-8')
        # 简化处理，假设直接从XML中解析出内容，实际需使用企业微信SDK进行解密和解析
        try:
            # 这是一个非常简化的示例，仅用于说明流程
            # 实际需要解析企业微信推送的XML消息体，提取用户消息
            # 例如使用 xml.etree.ElementTree 解析
            # 这里为演示，直接模拟一个用户消息
            user_message = "你好，请问公司最近的福利政策是什么？" # 假设解析后的用户消息
            
            print(f"收到来自企业微信的消息: {user_message}")
            
            response_text = call_deepseek_api(user_message, enterprise_knowledge)
            
            # 企业微信回复消息需要通过API调用发送
            # 获取 access_token
            token_url = f"https://qyapi.weixin.qq.com/cgi-bin/gettoken?corpid=YOUR_WECOM_CORP_ID&corpsecret={WECOM_SECRET}"
            token_resp = requests.get(token_url).json()
            access_token = token_resp.get("access_token")

            if access_token:
                send_msg_url = f"https://qyapi.weixin.qq.com/cgi-bin/message/send?access_token={access_token}"
                send_data = {
                    "touser": "@all", # 或者发送给特定用户或部门
                    "msgtype": "text",
                    "agentid": WECOM_AGENT_ID,
                    "text": {
                        "content": response_text
                    },
                    "safe": 0
                }
                requests.post(send_msg_url, data=json.dumps(send_data))
            
            return "success" # 必须返回 "success" 表示已成功接收
        except Exception as e:
            print(f"处理企业微信消息失败: {e}")
            return "error"

if __name__ == '__main__':
    # 提示用户创建 enterprise_knowledge.txt
    if not os.path.exists(KNOWLEDGE_FILE):
        with open(KNOWLEDGE_FILE, "w", encoding="utf-8") as f:
            f.write("公司最新的福利政策是：员工生日当天可获得额外带薪假一天。公司产品A的特点是高性能、低功耗。产品B适用于大数据分析场景。\n")
        print(f"已创建示例知识库文件: {KNOWLEDGE_FILE}。请编辑此文件以添加您的企业知识。")

    app.run(host='0.0.0.0', port=5000)
```

## 进阶与优化

1.  **真正的 RAG 知识库：** 对于大型知识库，简单地将整个文件作为上下文发送给 DeepSeek 是不可行的（有 token 限制）。真正的 RAG（Retrieval-Augmented Generation）会结合向量数据库（如 Faiss, Milvus, Weaviate）和文本嵌入模型（如 DeepSeek Embeddings）来实现。当用户提问时，先从向量数据库中检索最相关的几段知识，再将这些片段作为上下文传递给 DeepSeek 模型。
2.  **多轮对话与记忆：** 当前示例是单轮问答。要实现多轮对话，你需要存储用户的历史对话记录，并在每次调用 DeepSeek API 时将其作为上下文的一部分发送。
3.  **错误处理与日志：** 增加更健壮的错误处理机制和详细的日志记录，便于问题排查。
4.  **安全加固：** 对于生产环境，务必对 Webhook 地址进行严格的 IP 限制或签名验证，防止未经授权的访问。企业微信的加密消息处理是必须实现的。
5.  **自定义 DeepSeek 模型：** DeepSeek 提供了 `deepseek-coder` 等模型，可根据场景选择更合适的模型。
6.  **可视化管理界面：** 为知识库提供一个简单的 Web 界面，方便管理员增删改查知识内容。
