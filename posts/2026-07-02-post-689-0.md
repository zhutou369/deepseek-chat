## DeepSeek 实践：逐步构建你的定制食谱提示词

构建高质量的提示词（Prompt）是获得满意结果的关键。我们将通过迭代的方式，逐步细化你的指令。

### 步骤 1：基础指令

首先，给 DeepSeek 一个明确的任务方向。

```
请为我制定一份为期七天的低脂健康食谱。
```

### 步骤 2：加入详细约束

将你在“准备工作”阶段梳理好的信息，逐一添加到提示词中。越具体越好。

**示例提示词：**

```
请为我制定一份为期七天的低脂健康食谱，目标是减脂。
我的身高175cm，体重75kg，30岁男性，每周进行3-4次中等强度健身训练。
每日总热量估算在1800-2000卡路里之间，蛋白质占比30-35%，脂肪占比20-25%，碳水化合物占比45-50%。
我偏爱鸡胸肉、鱼肉（三文鱼、鳕鱼）、豆腐和各种绿叶蔬菜。不吃猪肉和红肉，对乳制品不耐受。
每天需要包含早餐、午餐、晚餐以及一份下午加餐。
所有餐点烹饪时间最好控制在30分钟以内，易于准备。
```

### 步骤 3：指定输出格式

为了让食谱更具实用性，你可以要求 DeepSeek 以结构化的方式呈现结果，并包含具体的细节。

**在上述提示词基础上，追加：**

```
请以表格形式呈现食谱。表格应包含以下列：
1.  日期/餐次
2.  菜品名称
3.  主要食材及估算分量（例如：鸡胸肉150g，西兰花200g）
4.  简要制作方法
5.  估算卡路里
6.  估算蛋白质含量
确保食谱每天不重复，并给出总体的每周食材采购清单。
```

### 步骤 4：迭代与优化

DeepSeek 首次生成的食谱可能不是完美的。仔细审阅后，你可以提出修改意见，进行迭代。

**常见迭代场景：**

*   **多样性不足**：
    `第一天的午餐和第三天的午餐菜品过于相似，请替换第三天的午餐，提供一个以鱼肉为主的新选择。`
*   **口味调整**：
    `某些菜品看起来比较清淡，能否在保持低脂的前提下，增加一些香料或天然调味料的建议？`
*   **烹饪细节**：
    `请为第二天的晚餐菜品提供更详细的烹饪步骤，包括调味料的用量。`
*   **食材替换**：
    `我发现我买不到藜麦，能否将所有用到藜麦的菜品替换成糙米或红薯？`

通过这种持续的对话和指令细化，DeepSeek 就能生成一份高度个性化且实用的周食谱。

### 2. Python 代码示例

以下是一个使用 Python 调用 DeepSeek API 生成食谱的示例代码。你需要先安装 `openai` 库：`pip install openai`。

import os
from openai import OpenAI

# 设置 DeepSeek API Key 和 Base URL
# 建议将 API Key 存储为环境变量，而非直接硬编码在代码中
DEEPSEEK_API_KEY = os.getenv("DEEPSEEK_API_KEY", "YOUR_DEEPSEEK_API_KEY_HERE")
DEEPSEEK_BASE_URL = "https://api.deepseek.com/v1" # DeepSeek API 的基础URL

# 初始化 DeepSeek 客户端
client = OpenAI(
    api_key=DEEPSEEK_API_KEY,
    base_url=DEEPSEEK_BASE_URL,
)

def generate_meal_plan(user_preferences: str):
    """
    根据用户偏好生成低脂健康周食谱。

    Args:
        user_preferences (str): 包含所有饮食需求与约束的详细描述。

    Returns:
        str: DeepSeek 生成的食谱文本。
    """
    messages = [
        {"role": "system", "content": "你是一位专业的营养师和食谱设计师，擅长根据用户需求制定低脂健康的个性化周食谱，并提供详细的食材、烹饪方法及营养估算。"},
        {"role": "user", "content": f"""
        请根据以下要求，为我制定一份为期七天的低脂健康食谱。

        {user_preferences}

        请以表格形式呈现食谱。表格应包含以下列：
        1.  日期/餐次
        2.  菜品名称
        3.  主要食材及估算分量（例如：鸡胸肉150g，西兰花200g）
        4.  简要制作方法
        5.  估算卡路里
        6.  估算蛋白质含量

        确保食谱每天不重复，并给出总体的每周食材采购清单。
        """}
    ]

    try:
        response = client.chat.completions.create(
            model="deepseek-chat",  # DeepSeek 的聊天模型名称
            messages=messages,
            temperature=0.7,      # 控制生成文本的创造性，0.7是一个不错的平衡点
            max_tokens=2000,      # 限制输出的最大长度
        )
        return response.choices[0].message.content
    except Exception as e:
        return f"生成食谱时发生错误: {e}"

# 你的详细用户偏好描述
my_user_preferences = """
我的身高175cm，体重75kg，30岁男性，每周进行3-4次中等强度健身训练。
目标是减脂，每日总热量估算在1800-2000卡路里之间，蛋白质占比30-35%，脂肪占比20-25%，碳水化合物占比45-50%。
我偏爱鸡胸肉、鱼肉（三文鱼、鳕鱼）、豆腐和各种绿叶蔬菜。不吃猪肉和红肉，对乳制品不耐受。
每天需要包含早餐、午餐、晚餐以及一份下午加餐。
所有餐点烹饪时间最好控制在30分钟以内，易于准备。
"""

# 调用函数生成食谱
meal_plan = generate_meal_plan(my_user_preferences)
print(meal_plan)

### 关键点解析：

*   **`DEEPSEEK_API_KEY` 和 `DEEPSEEK_BASE_URL`**: 务必替换为你自己的 API Key，并确保 `base_url` 指向 DeepSeek 的 API 入口。
*   **`messages` 结构**: `messages` 是一个列表，包含一系列字典。每个字典代表一个“角色”（`role`）的发言，如 `system`（设定AI的身份和行为）和 `user`（用户的输入）。
*   **`model`**: 指定你想要调用的 DeepSeek 模型，`deepseek-chat` 是一个常用的聊天模型。
*   **`temperature`**: 控制生成文本的随机性。值越高，输出越有创意；值越低，输出越集中和确定。对于食谱这类需要具体信息的任务，0.7是一个不错的起点。
*   **`max_tokens`**: 限制 DeepSeek 返回的文本长度，防止过长的响应。
