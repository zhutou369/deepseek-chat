## 准备工作：明确你的饮食目标与偏好

在向 DeepSeek 提出请求之前，花几分钟时间整理你的个人信息和饮食需求，这会是 DeepSeek 准确生成食谱的关键。信息越详细，食谱就越符合你的实际情况。

你需要考虑以下几个方面：

1.  **核心目标：** 你希望通过这份食谱达到什么目的？
    *   **减脂：** 控制总热量，降低脂肪摄入，提高蛋白质。
    *   **增肌：** 保证足够的热量和蛋白质摄入。
    *   **维持健康体重：** 均衡膳食。
2.  **基本信息与活动水平：** 这些数据有助于 DeepSeek 估算你的基础代谢和总能量消耗（TDEE），从而调整食谱的热量。
    *   性别、年龄、身高、体重。
    *   日常活动水平（久坐、轻度运动、中度运动、重度运动）。
    *   每周健身频率和类型（例如，每周3次力量训练，2次有氧）。
3.  **饮食偏好与限制：**
    *   **忌口/过敏：** 任何不能吃的食材（如猪肉、牛肉、海鲜、乳制品、麸质、坚果等）。
    *   **不喜欢吃的食材：** 即使不过敏，但个人不喜欢吃的食材。
    *   **素食/纯素食：** 如果你是素食者，请明确是蛋奶素、奶素还是纯素。
    *   **口味偏好：** 清淡、重口味、少盐少油等。
4.  **烹饪条件：**
    *   **烹饪时间：** 每餐能接受的准备和烹饪时长。
    *   **烹饪技能：** 简单烹饪、中等难度、复杂料理。
    *   **可用设备：** 烤箱、空气炸锅、蒸锅等。
5.  **餐次安排：**
    *   一日三餐？是否需要加餐（上午、下午、睡前）？

将以上信息整理成条理清晰的列表或段落，可以大大提高 DeepSeek 理解的准确性。

### 方法二：通过 DeepSeek API 进行程序化调用 (可选，高级用户)

如果你是开发者，或者希望将食谱生成功能集成到自己的应用或自动化流程中，DeepSeek API 是一个强大的选择。

1.  **获取 API Key：** 在 DeepSeek 平台上注册并获取你的 API Key。
2.  **选择编程语言和库：** 通常使用 Python 及其 `requests` 库或官方 SDK。
3.  **构建 API 请求：** 将你的 Prompt 放入 API 请求体中。

    **Python 代码示例：**

    ```python
    import requests
    import json

    # 替换为你的 DeepSeek API Key
    DEEPSEEK_API_KEY = "YOUR_DEEPSEEK_API_KEY" 
    API_URL = "https://api.deepseek.com/chat/completions" # 请根据DeepSeek官方文档确认最新的API地址

    # 你的食谱生成Prompt
    prompt_message = """
    请为我制定一份低脂健康的七日（周一至周日）食谱。
    我的目标是减脂，女性，28岁，身高165cm，体重60kg，每周进行4次中等强度有氧运动。
    我不吃牛肉，对花生过敏。偏好口味清淡，每餐烹饪时间不超过25分钟。
    食谱需包含早餐、午餐、晚餐，并可包含一次上午加餐。
    请确保食谱营养均衡，提供每餐的食材列表和简要烹饪方法。
    请以表格形式呈现，表格包含列：日期、餐次、菜品名称、主要食材、烹饪建议。
    """

    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {DEEPSEEK_API_KEY}"
    }

    data = {
        "model": "deepseek-chat", # 或其他DeepSeek模型，请查阅API文档
        "messages": [
            {"role": "system", "content": "你是一位专业的营养师和健康饮食规划师。"},
            {"role": "user", "content": prompt_message}
        ],
        "stream": False # 如果需要流式输出，设置为True
    }

    try:
        response = requests.post(API_URL, headers=headers, json=data)
        response.raise_for_status() # 检查HTTP响应状态码
        result = response.json()

        # 解析 DeepSeek 的响应
        if 'choices' in result and result['choices']:
            generated_content = result['choices'][0]['message']['content']
            print("DeepSeek 生成的周食谱：\n", generated_content)
            # 你可以进一步处理 generated_content，例如保存到文件或数据库
        else:
            print("DeepSeek API 响应中没有内容。", result)

    except requests.exceptions.RequestException as e:
        print(f"API 请求失败：{e}")
    except json.JSONDecodeError:
        print("API 响应不是有效的 JSON 格式。")
    ```
    **注意：** 请务必查阅 DeepSeek 官方 API 文档，以获取最新的 API 地址、模型名称和请求参数。

## 优化与迭代：让食谱更贴合你的需求

DeepSeek 生成的食谱只是起点，通过进一步的交互，你可以让它变得更加实用和个性化。

    ```

    ```

2.  **多样性与替代方案：** 为了避免饮食单调，或者应对某些食材的临时短缺，你可以请求 DeepSeek 提供替代方案。

    ```
    对于食谱中的鸡胸肉，能否提供两种低脂的鱼类或豆制品作为替代选项？
    ```

3.  **烹饪复杂性调整：** 如果你发现烹饪步骤过于复杂或过于简单，可以要求 DeepSeek 进行调整。

    ```

    ```

4.  **生成购物清单：** 这是 DeepSeek 另一项非常有用的功能，可以直接根据食谱生成一周的购物清单。

    ```
    请根据这份周食谱，生成一份详细的购物清单，按食材类别进行分类（例如：蔬菜、肉类、谷物、调料）。
    ```

5.  **适应特殊情况：** 如果你有出差、聚餐或其他特殊日程，可以提前告知 DeepSeek，让它调整或提供应对建议。

    ```
    我周四中午有商务午餐，无法自己烹饪。请建议我在餐厅如何选择低脂健康的菜品。
    ```
